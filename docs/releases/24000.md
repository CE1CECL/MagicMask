## 2022.1.26 MagicMask v24.0

It has been a while since the last public release, long time no see! A personal update for those unaware: I am now working at Google on the Android Platform Security team. Without further ado, let's jump right into it!

### MagicMaskHide Removal

I have lost interest in fighting this battle for quite a while; plus, the existing MagicMaskHide implementation is flawed in so many ways. Decoupling MagicMask from root hiding is, in my opinion, beneficial to the community. Ever since my announcement on Twitter months ago, highly effective "root hiding" modules (much **MUCH** better than MagicMaskHide) has been flourishing, which again shows that people are way more capable than I am on this subject. So why not give those determined their time to shine, and let me focus on improving MagicMask instead of drowning in the everlasting cat-and-mouse game 😉.

### Sunsetting MagicMask-Modules-Repo

Due to lack of time and maintenance, the centralized MagicMask-Modules-Repo was frozen, and the functionality to download modules from the repo is removed in v24.0. As a supplement, module developers can now specify an `updateJson` URL in their modules. The MagicMask app will use that to check, download, and install module updates.

### Introducing Zygisk

Zygisk is **MagicMask in Zygote**, the next big thing for MagicMask! When this feature is enabled, a part of MagicMask will run in the `Zygote` daemon process, allowing module developers to run code directly in every Android apps' processes. If you've heard of [Riru](https://github.com/RikkaApps/Riru), then Zygisk is inspired by that project and is functionally similar, though the implementation is quite different internally. I cannot wait to see what module developers can achieve using Zygisk!

### Documentation

For developers, details about `updateJson` and building Zygisk modules can all be found in the updated [documentation](https://topjohnwu.github.io/MagicMask/guides.html#magicmask-modules).
